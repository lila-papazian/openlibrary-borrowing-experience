# Rethinking the OpenLibrary Borrowing Experience

*A public Design Engineering case study.*

This repository documents the complete Design Engineering process behind rethinking the borrowing experience of OpenLibrary.

Rather than focusing on a visual redesign, this project explores how complex lending states can be communicated clearly through interface design.

The goal is to understand the domain first, identify real user pain points, document design decisions, and build a working prototype backed by evidence instead of assumptions.

## Objectives

* Understand how OpenLibrary's lending system actually works.
* Collect and synthesize evidence from real users.
* Model the borrowing domain and system states.
* Design interactions that reduce ambiguity and improve trust.
* Prototype the proposed solution in code.
* Document every major design decision and its tradeoffs.

## Principles

* Research before design.
* Evidence over assumptions.
* Problems before solutions.
* Decisions should be explicit and reviewable.
* Documentation is part of the product.

## Project Rules

This repository follows one simple rule:

> Don't design solutions until full understanding of the problem.

Every major decision should be traceable to evidence collected during research.

If a decision cannot be justified, it should be reconsidered.

## Why this repository?

This is not intended to be another redesign case study.

The objective is to document the reasoning behind every decision as if this work were happening inside a real product team, making the process as valuable as the final prototype itself.
